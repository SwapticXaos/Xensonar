# Machine Room Shift Plan v2

## Ziel dieser Phase

Die Verschiebung der Zuständigkeiten soll nicht nur theoretisch benannt, sondern im laufenden System sichtbar und sanft geführt werden.

- **Xensonar / Room V** bleibt Aufführungs- und Routingkern.
- **Forge 2** trennt seine beiden Arbeitsflächen bewusster:
  - `Wave / Material`
  - `Microtonal Logic`
- **Myzel / III.1** zeigt explizit gruppenbezogene Post-FX-Wege, ohne Erzeugungszuständigkeiten zurück in den Monolithen zu ziehen.

## Sanfte Verschiebung statt Bruch

Die Kern-App hält aktuell eine funktionale Balance. Deshalb werden Zuständigkeiten nicht brutal herausgerissen, sondern schrittweise nach außen verlagert.

### Forge 2

Die beiden Arbeitsflächen sollen klarer unterscheidbar werden, teilen aber weiterhin:

- BPM-/Takt-Deutung
- Exportvertrag
- Materialname / Rolle
- Adapterbrücke

So bleibt die Außensprache stabil, während innen die Spezialisierung wächst.

### Myzel

Myzel bleibt nachgelagert. Sichtbare Gruppenpfade sind:

- Master
- Partikel
- Drone
- Wellenstarter
- Forge-Material

Wichtig: Diese Pfade zeigen Fokus und spätere Routing-Intention, ohne Segmenterzeugung oder Loop-Bau heimlich zurückzuziehen.

## Drift-Prävention

Nach Chat-Resets oder größeren Patch-Neuansätzen zuerst prüfen:

1. Artefakt-Hygiene
2. Adapter-/Typvertrag
3. Begriffe der Maschinenräume
4. Persistenz von Fokus-/Arbeitsflächenzuständen

So werden Frühphasen-Altlasten nicht wieder unbemerkt dominant.
